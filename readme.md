<h1 align="center">🎙️ Automatic Announcement Quality Analyzer (AAQA)</h1> <p align="center"> <b>A Digital Signal Processing (DSP) based system for real-time evaluation of public announcement audio quality.</b> </p> <p align="center"> <a href="https://aaqa-8furfb750-rishav-dev1.vercel.app/" target="_blank"> <img src="https://img.shields.io/badge/View%20Live%20Report-%23007ACC?style=for-the-badge&logo=vercel&logoColor=white" /> </a> </p> <hr> <h2>👥 Team Members</h2> <ul> <li><b>Abhinav Kesarwani</b> (23AI002)</li> <li><b>Ajitesh Channa</b> (23AI006)</li> <li><b>Anant Jain</b> (23AI007)</li> <li><b>Rishav Kumar</b> (23AI043)</li> </ul> <hr> <h2>📌 Project Overview</h2> <p> The <b>Automatic Announcement Quality Analyzer (AAQA)</b> is a non-intrusive audio quality evaluation system designed using <b>Digital Signal Processing (DSP)</b> techniques. It analyzes public announcement audio in real-time and provides objective quality assessment without requiring a reference signal. </p> <p> The system is specifically tailored for <b>transportation infrastructure</b> such as railway stations and airports, where clarity and intelligibility of announcements are critical for safety and operational efficiency. </p> <hr> <h2>🚀 Key Features</h2> <ul> <li>🎧 Audio Upload & Live Recording Interface</li> <li>📊 Real-time Quality Scoring (0–100 scale)</li> <li>📈 Detailed DSP Metric Visualization</li> <li>🧠 Non-Intrusive (Blind) Audio Quality Evaluation</li> <li>📝 Automatic Speech Transcription</li> <li>📉 Noise Detection & Clarity Feedback</li> </ul> <hr> <h2>⚙️ Technology Stack</h2> <h3>Frontend</h3> <ul> <li>Next.js</li> <li>React</li> <li>WaveSurfer.js</li> </ul> <h3>Backend</h3> <ul> <li>FastAPI</li> <li>Python</li> </ul> <h3>Signal Processing & Analysis</h3> <ul> <li>NumPy</li> <li>SciPy</li> <li>SpeechRecognition (Google API)</li> </ul> <h3>Experimental (Research Phase)</h3> <ul> <li>Wav2Vec2 (Transformers)</li> <li>PyTorch</li> </ul> <hr> <h2>🧠 Core DSP Concepts</h2> <ul> <li><b>Signal-to-Noise Ratio (SNR)</b> — Measures speech clarity against background noise</li> <li><b>Spectral Flatness</b> — Distinguishes tonal speech from noise-like signals</li> <li><b>Zero Crossing Rate (ZCR)</b> — Detects signal variability and distortion</li> <li><b>Formant Energy Ratio</b> — Evaluates speech energy concentration</li> </ul> <hr> <h2>📐 Scoring Methodology</h2> <p> The system employs a <b>non-linear scoring framework</b> to align with human auditory perception: </p> <ul> <li>Sigmoid transformation for SNR (perceptual scaling)</li> <li>Power-based weighting for formant energy</li> <li>Logarithmic scaling for spectral flatness</li> </ul> <p> A multiplicative scoring mechanism ensures that degradation in any feature significantly impacts the final quality score. </p> <hr> <h2>🏗️ System Architecture</h2> <pre> Audio Input → Preprocessing → DSP Feature Extraction → Scoring Engine → Output (Score, Label, Feedback, Transcription) </pre> <hr> <h2>📊 Output Parameters</h2> <ul> <li>🎯 Quality Score (0–100)</li> <li>📌 Classification (Excellent / Good / Poor / Bad)</li> <li>🔊 Noise Level Estimation</li> <li>🧾 Speech Transcription</li> <li>📈 Detailed DSP Metrics</li> </ul> <hr> <h2>⚠️ Limitations</h2> <ul> <li>Heuristic-based scoring (no adaptive learning)</li> <li>Requires environment-specific tuning</li> <li>Deep learning model not fully integrated</li> </ul> <hr> <h2>🔮 Future
